End to End Snowflake ELT Pipeline

Snowflake
About the Project

This project builds a production style ELT pipeline in Snowflake that ingests semi structured cricket JSON data and transforms it into a clean dimensional schema. Using Snowflake native tools like internal stages, COPY INTO, VARIANT parsing, and FLATTEN, it covers the full workflow from raw ball by ball data ingestion to a fully populated star schema.

Problem Statement

Raw cricket match data in JSON format is inaccessible to analysts and reporting tools in its native form. Without a structured pipeline, ball-by-ball records, match outcomes, player data, and delivery statistics cannot be queried or aggregated meaningfully. The challenge was to design a layered ELT architecture that could reliably ingest deeply nested JSON, parse and flatten multi-level arrays across innings, overs, deliveries, wickets, and fielders, enforce relational integrity across entities, and deliver a dimensional model ready for downstream cricket analytics.

Output Interpretation

Explain the final output and what it means. Help someone non-technical understand the end result.

  • Four-layer architecture spanning LAND, RAW, CLEAN, and CONSUMPTION schemas handling the full journey from local JSON ingestion to analytics-ready dimensional model
  • CLEAN schema parses complex nested JSON into three structured tables covering match details, player data, and ball-by-ball deliveries using multi-level lateral flattening
  • Primary and foreign key constraints enforced across all CLEAN tables to ensure relational integrity throughout the pipeline
  • CONSUMPTION schema houses a full star schema with six dimension tables and two fact tables capturing both match-level aggregates and ball-by-ball delivery granularity
  • End-to-end Snowflake-native implementation using internal stages, COPY INTO, VARIANT parsing, and FLATTEN demonstrating production-grade cloud data engineering applicable to real-world analytics infrastructure
