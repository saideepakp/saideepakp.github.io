End-to-End Data Engineering Project — FMCG Domain

Databricks AWS S3 PySpark Power BI
About the Project

This is an end-to-end data engineering project built on Databricks Free Edition, simulating a real-world retail acquisition scenario in the FMCG domain. A large sports retail company (parent — Atlikon) acquires a smaller sports nutrition company (child — Sports Bar), and the goal is to build a unified data pipeline that consolidates data from both companies into a single Lakehouse architecture. The project covers the complete data engineering lifecycle from raw data ingestion to a Power BI dashboard, making it a comprehensive demonstration of modern data engineering practices.

Problem Statement

When a parent company acquires a child company, their data systems are typically incompatible — different schemas, naming conventions, data formats, data quality standards and ingestion frequencies. Business teams need a unified view of both companies' data immediately after acquisition to make informed decisions. The challenge is to:

  • Consolidate historical data from both companies into a single source of truth
  • Handle significant data quality issues in the child company's data
  • Build a reliable day-to-day incremental pipeline for the child company's transactional data
  • Deliver a unified, analytics-ready dataset for business reporting via Power BI

Approach
  • Designed a three-layer Medallion Lakehouse architecture on Databricks with Bronze, Silver and Gold layers, treating both companies as separate data domains in Bronze and Silver and merging only at the Gold layer for maximum pipeline flexibility and independence
  • Ingested raw historical data for the parent company from Databricks Volumes and the child company from Amazon S3 into the Bronze layer as-is, then applied extensive Silver transformations including fixing typos, normalizing date formats, handling invalid identifiers with fallback values and generating consistent product codes using MD5 hashing
  • Aligned schemas and data types between parent and child Silver tables through a dedicated validation layer before merging into a unified Gold layer using unionByName, aggregating child company daily orders to monthly grain to match the parent company's data structure
  • Designed an incremental pipeline for child company day-to-day orders using a staging table pattern with monthly re-aggregation logic, ensuring Gold layer totals are always 100% accurate regardless of when new data arrives, with Delta Lake MERGE operations guaranteeing no duplicates across any layer
  • Orchestrated the full pipeline using Databricks Workflows with a historical job running parent and child pipelines in parallel and a daily scheduled incremental job, with the final unified Gold view connected to a Power BI dashboard delivering post-acquisition reporting across sales overview, product performance and customer analysis
